# Super Bowl LX Score Prediction Game — Project Log

**Date:** February 2026
**Type:** Vibe coding experiment — single-file HTML app

## Why
Built for fun and as a hands-on vibe coding exercise. Wanted to experience the full process firsthand and document the learnings.

## What It Does
A single-page web app where friends/family predict the Super Bowl LX final score (Seahawks vs Patriots). Features live score via ESPN API, automatic leaderboard ranking, confetti celebrations, and localStorage persistence.

## Timeline & Process
- **10-15 min** — Got the core app functional (prediction form, scoring logic, live API, leaderboard)
- **Minimal time** — Design polish and UI improvements
- **1-2 hours** — The real work: simulating actual gameplay, discovering edge cases, handling errors, and refining game rules through repeated testing

## Key Iterations (from git history)
1. Initial build — full functional app
2. Simplified UI & scoring algorithm
3. Added unsaved input confirmation on modal close
4. Prevented tie predictions (Super Bowl can't end in a tie)
5. Hid winner display at 0:0 score
6. Show leaderboard/rankings only when game has real scores

## Takeaway
Building something functional is fast. The real time investment is in **relentless simulation and iteration** — actually using the product, discovering UX gaps, and refining through real user scenarios. That's where the craft lives.
