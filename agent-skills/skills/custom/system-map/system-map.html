워크플로우 스킬 (8) — 일하는 방식

👤
user-research
페르소나, 저니맵, 경쟁사 분석, Anti-Persona
Discovery
프로토콜: Hypothesis → Define → Compete → Map → Insight
산출물: 페르소나, 경쟁사 3+개 분석, 저니맵
핸드오프: → brainstorm, design, quality
💡
brainstorm
발산-수렴 프레임워크, SCAMPER, Impact/Effort
Ideation
프로토콜: Frame → Diverge(최소 10개) → Cluster → Converge → Action
기법: 브레인덤프, 역전사고, SCAMPER, 유사매핑
산출물: Impact/Effort 매트릭스, Top 3, Quick Win
🎨
design
UI/UX 3-Agent 분석, 비주얼 계층, 반응형, Figma→Code
Design
프로토콜: Understand → Structure → 3-Agent(UX+Design+Brand) → Visual → Review
Figma: Deep Fetch(depth 6+) → Extract → Build with exact values → Verify
원칙: 센터레이아웃, shadcn 스타일, oklch, 터치우선
💻
code
코드 작성/리뷰 표준. 보안 체크리스트, 패턴 검증
Build
프로토콜: Analyze → Pattern → Implement → Self-Review → Refactor
체크리스트: XSS 방지, localStorage 패턴, CORS fallback, DOM focus 복원
Confidence: 🟢 테스트 통과→적용 / 🔴 미검증→금지
✅
quality
QA 시뮬레이션, 엣지 케이스, 📸 스크린샷 비주얼 검증
QA📸 Visual
프로토콜: Plan → Flow Test → Edge Cases → 📸 Visual Verify → Fix → Report
스크린샷: UI 변경 시 캡처→비교→재캡처 루프. GIF 녹화로 멀티페이지 리뷰
트리거: "이상해", "깨졌어", "안 맞아" → 자동 캡처
🚀
ops
배포/운영. Git 워크플로우, 헬스체크, 롤백 계획
Deploy
프로토콜: Assess → Build → Deploy → Verify → Document
규칙: 로컬 테스트 없이 배포 금지, force push 금지, 롤백 계획 필수
📊
data
데이터 분석/시각화. Chart.js, 인사이트 도출
Support
프로토콜: Define → Collect → Clean → Analyze → Insight
규칙: 데이터 없이 결론 금지, 출처+시점 명시
🔬
research
멀티 에이전트 병렬 리서치. 출처 검증, 반론 필수
Support
프로토콜: Scope → Assign(3+ 병렬) → Collect → Synthesize → Report
규칙: 출처 없는 주장 금지, 반론 필수, 학술지>기관>뉴스>블로그

슬래시 커맨드 (4개) — 복합 워크플로우 트리거

프롬프트는 에이전트를 스폰하고 스킬을 조합하는 매크로입니다.
스킬이 "방법론", 에이전트가 "실행자"라면, 프롬프트는 "시작 버튼"입니다.
🔍
/analyze-toolkit-gaps
세션 히스토리 분석 → 부족한 스킬/에이전트 식별 → 빌드 계획
researcher 스폰메타
트리거: "what skills do I need", "toolkit gaps", "audit my workflow"

워크플로우:
  1. Session Analysis — 세션 로그 분석, 반복 작업 클러스터링, 빈도×노력 순위
  2. Gap Analysis — 기존 도구 커버리지 체크 → 커뮤니티 검색 → Covered/Adopt/Extend/Build 분류
  3. Build Plan — 각 갭에 대해 구체적 스펙 (이름, 트리거, 워크플로우, 노력 추정)

산출물: 워크스트림 순위 + 갭 분석 테이블 + 구현 계획 + 우선순위
인자: $FOCUS — 선택적 분석 범위 (예: "WinUI work", "frontend")
🎯
/orchestrate
서브에이전트 기반 개발 — 작업 분할 → 병렬 위임 → 통합
서브에이전트병렬
트리거: 여러 파일/모듈에 걸친 대규모 작업

워크플로우:
  1. 작업 분석 + 의존성 그래프 생성
  2. 독립 작업 식별 → 서브에이전트에 위임
  3. 같은 파일 터치하는 작업은 순차 실행 (충돌 방지)
  4. 결과 통합 + 리뷰

가드레일: 같은 파일/인터페이스 터치 시 병렬 금지
인자: $TASKS — 수행할 작업 목록

트리거: UI 폴리싱, 비주얼 품질 개선

워크플로우 (최대 3회 루프):
  1. 프로젝트 파일에서 플랫폼 자동 감지 (Web/Windows/Mac/iOS)
  2. 타겟 윈도우/시뮬레이터 찾기
  3. 📸 "Before" 스크린샷 캡처
  4. 비주얼 이슈 분석 (레이아웃, 타이포, 대비, 플랫폼 규칙)
  5. 확실한 수정만 적용 (High confidence only)
  6. 빌드 → 📸 "After" 스크린샷 재캡처
  7. Before/After 비교 → 회귀 확인

지원 플랫폼: Web, Windows, macOS, iOS/watchOS
